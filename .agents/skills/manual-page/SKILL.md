# 手册页生产标准

## 定位

手册页属于四层学习体系的「参考手册层」：学习者从骨架卡或案例的卡住点跳进来查一节，读完就走。它不是课本，不追求可通读性，追求**查到的每一节都自带推导、数字可回源、读到哪里该停有边界**。

金标准：`output/m02.html`。动手前先完整读一遍它和 `Learn Plan - AI半导体投资.md` 的「学习方式」「案例质量标准」两节。

## 为什么有这份标准

这个项目的第一版模块页因为「只给概念和结论」被推翻，第二版「怎么判断」因为「规则没有出处和推导」被推翻。用户的核心要求：**结论必须当面从机制和数字推出来；规则必须有真实出处；数字必须可回源。**违反任何一条，整页返工。

## 生产流程（顺序执行，不可跳步）

### 第一步 · 查证（写任何 HTML 之前）

1. 列出本页拟使用的全部事实、数字、事件、规则的清单；
2. 逐条用 WebSearch 找一手来源：公司年报/10-K/6-K、财报电话会转录、官方统计（SEMI 等）、技术白皮书、官方公告。记录来源名+日期；
3. 一手找不到的：可靠专业研究（SemiAnalysis、TechInsights、CSET 等）作为「估算」，标注估算方与年份；仍找不到的**弃用**，不改写成模糊说法蒙混；
4. 查证结果与拟写内容矛盾时，服从证据（本项目曾因查无实据弃用过「某设备商与特定客户绑定」的说法——这是荣誉记录，不是事故）；
5. 凭记忆的数字一律不上页面，包括「大约」「据说」式的软化写法。

### 徽章可点击（强纪律，所有页面适用）

每一个徽章都必须是可点击的 `<a>`，点击直接跳向该条断言的核验源头，让读者一键从"我读到一个数字"到"我看到它的原始出处"。**span 徽章一律返工。**

语法：

```html
<!-- 优先：直链到一手来源 URL（年报、电话会转录、官方公告、SEMI 统计页等），新窗口打开 -->
<a class="badge badge-verified" href="https://…" target="_blank" rel="noopener">已核验 · ASML 2024 年报</a>

<!-- 无直链（专家判断、行业通行口径、无公开链接的估算等），锚点到页脚 .src 的对应条目 -->
<a class="badge badge-expert" href="#src-3">专家判断 · 通行口径</a>

<!-- 页脚来源清单：每个 <li> 必须带 id，便于徽章锚点定位 -->
<section class="src">
  <ol>
    <li id="src-1"><a href="https://…" target="_blank" rel="noopener">ASML 2024 Annual Report</a></li>
    <li id="src-2"><a href="https://…" target="_blank" rel="noopener">台积电 2024 年报</a></li>
    <li id="src-3">通行口径：设备行业常态交期 10-14 周（SEMI 沿用统计）</li>
  </ol>
</section>
```

规则：
- 徽章优先直链一手 URL；同一来源在页内出现多次可指向同一 URL。
- 无 URL 的（专家判断/无链接口径/自算估算）锚点到页脚 `.src` 的对应 `<li id="src-N">`。
- 页脚 `.src` 保留完整来源清单（一览所有出处），每个 `<li>` 必须有 `id`。
- 外链一律 `target="_blank" rel="noopener"`。
- 徽章文本仍写「已核验/专家判断/估算 · 来源名+日期」，颜色规则不变。

### 第二步 · 内容写作

页面骨架（与 m02 逐段对应）：

```
page-head：crumb（第X层·模块 N/14）→ h1 → core-q（核心问题）
验收题 note（max-width:860px）：3 道「学完必须能独立回答」的题，正文必须足以作答
concepts chips：本模块概念清单
tabs：本质 | 怎么判断 | 练习
.src：来源清单（全链接）
页脚：memo 能力框 → 标记完成 → 上一/下一模块
```

**本质页**：编号 h2 小节。写作红线：
- 每个结论当着读者的面推导出来。从公开数据算数用推导条 `.derive`（例：收入÷出货量=晶圆均价）；公式先讲直觉版再给式子；
- 比喻只做入口，进门后必须讲真实机制；
- 具体数字必须是查证过的真实量级，禁止「示意数字」；
- 善用跨模块预告：本页机制是后续哪个模块的地基，点一句（如「模块 9 的瓶颈铁律直接建在这上面」），给读者结构感；
- 结尾放「不要学太深」note：列出本页要带走的、能生成判断的几个模型，并划出投资够用线。

**怎么判断页**——固定由三部分组成，顺序和格式照 m02：

（一）开头声明段：一句话，样式为 muted 小字，模板——「这一页只交付 N 件真实存在的工具。每件都按同一结构给齐：来源 → 为什么成立 → 真实案例 → 怎么操作 → 失效边界。找不到出处、无法用真实案例验证的『规则』，不会出现在这里。」

（二）工具若干件：**宁少勿多**（m02 是三件），每件都要写深。格式：
- 标题：h2「工具N：名称」，名称用一句人话点出它是什么（如「交期（lead time）是供需的温度计」「设备订单领先芯片收入」），不用抽象名词；
- 正文：一个 `.chain`，五行，`.chain-row` 的 `<b>` 标签固定用这五个词——**来源 / 机制 / 案例 / 操作 / 失效**（失效行加 `chain-fail` 类）：
  1. **来源**——谁在用、在哪被记录（管理层原话/行业指标沿革/具名机构实践），可核验并带徽章；指标停发、口径变更这类沿革本身写进来源；
  2. **机制**——从本质页已建立的机制推出这个信号为什么含信息、领先性从哪来；
  3. **案例**——已核验的真实事件带着当时的数据完整走一遍（起点→过程→拐点→后来发生了什么），并说明这个工具的正确用途（如「确认周期位置，不是抄顶逃底的秒表」）；可交叉引用练习页的决策复盘避免重复；
  4. **操作**——数据在哪个公开源、多久更新、怎么读、和什么交叉验证；由机制推导的对照表/翻译表放在这一段并标注「由XX推导」；
  5. **失效**——什么时候骗人，逐条编号（①②③），尽量配真实反例。
- 一件不齐就删掉这条工具。自创流程不得冒充行业工具：要么并入某条工具的「操作」段，要么明确标注「由本页机制推导」。

（三）结尾固定一节：h2「操作清单：判断一个XX标的」。先一句 muted 引言说明「以下每项由本页机制直接推导，是工具的落地检查位，不是独立工具」；然后 `.checklist`，每项 = 加粗的问句 + 括号注明依据（来自哪件工具或本质页哪一节，跨模块的注明模块号）。

**练习页**——三段固定顺序，标题规范照 m02：
- 决策复盘：h2「决策复盘N · 日期：一句话事件」；`.case`（data-case）：真实事件+日期+已核验徽章 → 学习者先选（按钮，选项不打分）→ 展开专家推理链 → 必须有「诚实边界」行（区分推理质量与结果运气，说明推理对不保证股价）；
- 迁移题：h2 用「迁移题（模块里没有现成答案，用机制自己推）」；`.scenario`（data-reveal），展开后给推理对照而非标准答案，标注「警惕」行；
- 自测：h2 用「自测（算得出来才算会）」；`.quiz`（data-quiz）优先计算型，data-explain 必填且要讲清为什么；
- 末尾「本模块要带走的 memo 能力」note。

### 第三步 · 交互组件

组件分两类，都要做，用途不同：

**A 类 · 直观化组件**——使命是把晦涩变直观。写本质页时先列「晦涩点清单」：哪些内容要求零基础读者在脑中自行想象空间结构、时间流程、动态反馈或数量级差距？为每个高价值晦涩点配一个组件。判断标准：**操作或观看它十秒获得的直觉，超过读三段文字**；达不到就不做。形态工具箱：
- 流程步进动画（工艺循环、传导链——可步进、可自动播放）
- 分层解剖 / 爆炸图（加速卡、封装结构——点击逐层展开高亮）
- 参数化过程模拟（晶圆撒缺陷、牛鞭波动传播、木桶水位——拖参数看系统响应）
- 数量级对比（价格、投资额、时间尺度的可视对比）
- 时间轴回放（指标曲线与后续事件对照）

**B 类 · 推理组件**——计算器、真实数据图、决策复盘。参数必须是查证过的真实值并标注来源。

两类共同纪律：
- 每个组件配一句「看什么」引导语，告诉读者应该从中看出哪个结论；
- 不做文字的装饰性重复——组件必须呈现文字无法高效表达的东西；
- 教学性简化必须标注（如「容错模型为教学简化」）；
- 纯原生 SVG/CSS/JS，无外部依赖；动画尊重 prefers-reduced-motion；键盘可达；元素控制在千级以内，输入频繁时用 requestAnimationFrame 节流。

金标准：m02 的晶圆缺陷模拟（A 类）与爬坡计算器（B 类）。

技术速查：
- 公共脚本 `assets/app.js` 提供：导航渲染（新页面要把 LAYERS 里对应项 `built: true`）、`data-tabs`、`data-reveal`（配 `data-reveal-group` 互斥、`data-open-text` 换文案）、`data-quiz`；`APP.svgEl / showTip / hideTip / fmt / fmt1`；
- `data-case`（先选后展开）目前需页内脚本实现，照抄 m02 的通用写法；
- 样式类：`.derive`（推导条）、`.chain`（五件套/推理链）、`.case`、`.scenario`、`.quiz`、`.note`、`.ctl-panel/.ctl`（滑块）、`.viz`（图表容器）、`.checklist`、`.badge-*`、`.src`；
- 图表规范（dataviz）：单轴；线 2px 圆角；柱 ≤24px、数据端 4px 圆角、基线平角；网格 hairline 实线；文字用墨色不穿数据色；直接标签节制；hover 提示 + 数据不全标注时给表格视图；系列色用已验证的 `--viz-s1..s4` 蓝阶，状态/瓶颈用琥珀 `--accent`；
- 正文与卡片统一 860px 对齐；`body data-page="mXX"` 必须正确；aria 与键盘可达照 m02。

## 语言基调

像一个思路清楚的从业者给聪明的外行讲话：直接、具体、无废话。允许长段落，不允许空段落。「无缝」「赋能」「闭环」「沉淀」「对齐」这类词一律不用。
