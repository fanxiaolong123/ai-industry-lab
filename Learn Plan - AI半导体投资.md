# Learn Plan - AI半导体投资

## 本文件用途（先读这里）

本文件是学习 AI 半导体投资的教练文档。内容自成一体，包含：学习合同、四层学习体系与微观学习法、知识地图、案例与实盘的质量标准。

配套资产：本地交互知识库 `output/spine01.html` 起（骨架卡 15 张 + 参考手册 14 篇的正式版本，含计算器和决策复盘）。骨架卡与手册页的定稿以 HTML 为准；本文档保留结构性的教练标准与知识清单。

## 学习合同

起点：对 AI 行业尤其是制造业部分几乎没有体感；能看到 GPU、HBM、CoWoS、CapEx、估值等词，但无法理解它们之间的物理关系、产业前史和投资含义。

目标：从零基础建立 AI 半导体投资研究能力，最终能独立研究一个美股、A 股、港股或台韩日相关标的，并写出自己的投资 memo。

最终能力标准：
- 能判断标的处在 AI 半导体产业链哪一层。
- 能解释 AI 需求如何传导到该公司收入和利润。
- 能区分真实订单、真实瓶颈、市场映射和伪叙事。
- 能用四铁律判断瓶颈、垄断、库存弹性和扩产 PTSD。
- 能判断周期位置、估值是否 price-in、什么证据会证明自己错了。
- 能形成仓位、持有周期、退出条件和复盘标准。

## 学习方式：四层体系

学习主线不是按模块顺序阅读。四层结构按时间展开：

1. **骨架层**（一次性，几小时）：15 张核心模型卡，先全局过一遍，给一切细节搭架子。
2. **案例主线**（数周，训练主体）：约 12 个真实决策复盘，一个一个做；知识由案例拉动，卡住才去查手册。
3. **参考手册**（不通读，按需查）：14 篇模块页 + 数据源索引；由骨架卡和案例的「卡住点」跳入。
4. **实盘练习**（长期，体感唯一来源）：跟真实财报季，写 memo，季度对账。

配套：间隔复习队列——骨架卡与案例结论进队列，每天 10 分钟凭记忆自测。

### 微观学习法（单个知识点怎么学会）

- 知识点三分法：现在必须懂（骨架卡）／遇到再懂（案例拉动）／永远不用懂（超过「投资够用线」的工程深度）。
- 学会一个点的三个动作：读白话版 → 合上页面用自己的话复述 → 用它推一个没见过的问题。
- 掌握判据（三条全到才算）：能讲给外行听懂；能在新信息里认出它；能用它算粗数。
- 卡住三步：把看不懂的原句发给 AI 老师要搭桥解释 → 回到对应模块查上下文 → 标记跳过，等案例里再遇到。
- 每日节奏（30-60 分钟）：10 分钟凭记忆复述昨日内容 → 30 分钟新内容 → 10 分钟写三句话总结并接受点评。

### 骨架卡清单（15 张）

1. 供给慢、需求快：扩产 24-36 个月，一切周期现象的根源
2. 固定成本主导：涨价与利用率的利润弹性是跳变不是线性
3. 良率 = 成本曲线：爬坡斜率就是毛利率的日历
4. 出货 = 最短板：瓶颈环节拿走定价权，且瓶颈会漂移
5. 渠道逐层放大订单波动：缺货本身是风险信号
6. 库存结构决定波动结构：直供品类平缓，商品化品类最凶
7. 需求分两种：换机式（周期）与范式革命（台阶）
8. 技术代际是生死：掉队瞒不住，掌门人押注决定命运
9. 生态分工降低设计门槛；工艺-设计耦合越深，周期越凶
10. 需求传导链：模型公司→云资本开支→芯片订单→制造封装→设备材料→电力；信号硬度逐级升、前瞻性逐级降
11. 领先指标都来自扩产时序：设备订单、交期、测试设备
12. 价格是滞后读数：先行量是交期、长约、预付款
13. 市场定价在标题层，超额认知在流程层
14. 产业刚开始 ≠ 股价便宜：产业周期与股价周期错位
15. 规则必须五件套齐全：来源、机制、案例、操作、失效

### 案例主线：质量标准与生产流程

质量目标：每个案例是对一次真实投研决策的忠实重建，推理链达到专业 memo 水准；案例训练的就是模块 14 memo 模板的缩微版。

案例六要素（缺一不发布）：
1. 真实事件、真实时点：判断点之前的信息集只含当时可得的信息，严防事后信息泄漏；
2. 完整专业推理链：定位（产业链哪层）→ 机制拆解 → 数量化（真实数字算账）→ 反方观点 → 可证伪的判断与验证信号；
3. 已知结局对照：事后走势与归因；
4. 过程与结果分离：明确「推理对但可能结果错」的运气成分，反之亦然；
5. 学习者动作点不少于两处：先下判断、中途动手算；
6. 资料底稿：全部来源带日期、可点击、可审计。

生产流程：选题 → 一手资料采集（全部标日期）→ 时间线重建 → 信息集切割（判断点前/后）→ 推理链写作 → 反方与失效分析 → 事后对照 → 按验收清单逐条质检后发布。

验收清单：①关键数字全部一手来源+日期；②判断点前信息集无事后信息；③推理链五要素齐全；④含「如果错了会是什么样」；⑤结局归因区分推理质量与运气；⑥动作点≥2；⑦底稿可审计。

首批案例池：已核验三个（台积电 N3 稀释 2024、ASML→KLA 传导 2026-01、英特尔 18A 爬坡 2026）；待建：2020-2022 缺货与骤然逆转、存储 2023 减产到 2026 紧缺、量跌价涨的利润结构（存储厂季报）、云厂商资本开支拐点识别、映射类炒作辨伪（A 股）等。

能力边界（诚实声明）：案例训练推理链；信息搜集的耐力和真金白银的心态只能由实盘层训练，案例不假装能替代。

### 实盘练习：操作方案

阶段 0 · 准备（骨架卡 + 前三个案例完成后即可开始）：
- 选 1-2 家跟踪对象（披露质量高、处在骨架模型覆盖的环节、财报季临近；建议至少一家美股，电话会转录免费公开）；
- 建立判断日志，固定字段：日期／我的判断／依据／反方／验证信号／到期检查日。

阶段 1 · 跟一个完整财报季：
- 财报前：写预判卡——预计管理层会怎么说交期、良率口径、资本开支语气，写成可证伪的具体预期；
- 财报日：读转录，用工作表逐句标注「叙事」还是「证据」；
- 48 小时内：写半页 mini-memo（定位/机制/数字/反方/验证信号/结论强度），提交点评；
- 下季度：翻出判断日志逐条对账——错的归因到事实错、机制错还是时点错。

阶段 2 · 第二个财报季：用模块 14 的 12 条模板写完整 memo（含 kill criteria 与纸上仓位），三个月后复盘，开始积累个人错误清单。

阶段 3 · 真金白银与否是学习者自己的决定；前提是至少两个财报季的纸上记录加错误清单，且执行仓位×策略匹配原则。

反馈来源三条：AI 老师点评（快）、管理层下季兑现对照（中）、市场结果（慢）。时间预算：财报季每周 2-3 小时，平时每周 1 小时（用骨架卡翻译当周新闻）。

目标：先建立行业直觉。没有这层，后面的 HBM、CoWoS、CapEx、LTA、良率、库存弹性都会只是术语。

### 模块 1：AI 为什么会变成硬件和制造业问题

核心问题：AI 不是软件吗，为什么最后投资人天天盯 GPU、HBM、CoWoS、存储、电力和液冷？

必须懂的概念：
- token
- 训练
- 推理
- GPU
- 数据中心
- 云厂商 CapEx
- AI cloud
- agent 对推理需求的放大

案例：
- OpenAI / Anthropic 需求如何变成云厂商采购。
- Microsoft Azure 增长如何影响上游 AI infra。
- NVIDIA Data Center 收入为何是 AI 硬件需求的直接信号之一。

练习题：
- 如果一条新闻说“某模型推理成本下降 80%”，这对 GPU、HBM、电力、云厂商分别可能是利好还是利空？
- 如果一家公司宣布建设 AI data center，你要问哪五个问题判断它是不是会拉动上游订单？

Memo 输出能力：
- 能写清楚“AI 需求如何传导到标的公司”这一段。

- 沉积
- 清洗
- 量测

案例：

- 为什么 KLA 量测会在先进节点中变重要。
- 为什么 Intel 18A 的良率爬坡会影响代工可信度。

练习题：
- 一家公司说“良率每月提升 7%-8%”，这句话为什么对投资人重要？
- 台积电增加 EUV 设备订单，除了 ASML，还可能影响哪些设备环节？

Memo 输出能力：

### 模块 3：封装、测试与系统集成 101

练习题：

### 模块 5：半导体产业链总地图

核心问题：半导体行业有哪些层，每一层靠什么赚钱，风险来自哪里？

必须懂的概念：
- 设计
- 制造
- 封装
- 设备
- 材料
- 存储
- 光模块
- 电力液冷
- fabless
- foundry
- IDM
- ARM 与 X86
- SoC
- IP 生态
- 工艺-设计耦合度

案例：
- NVDA、AMD、AVGO 是设计逻辑。
- TSMC、Intel Foundry、Samsung、中芯国际是制造逻辑。
- ASML、KLAC、LRCX、AMAT、北方华创是设备逻辑。
- 日月光、长电、通富、台积电先进封装是封装逻辑。
- SoC 集成现成 IP，生态分工大幅降低设计门槛，但“有”不等于“做得好”。
- 存储的工艺与设计强耦合、只能一体化经营，这决定了它的周期远比可分工的设计环节凶。

练习题：
- 给你一个陌生公司，如何用 10 分钟判断它在产业链哪一层？
- 同样是 AI 受益，为什么设备公司和设计公司的估值方式不同？
- 为什么设计类公司常“一步一台阶”，而存储公司常暴涨暴跌？

Memo 输出能力：
- 能写出“公司产业链位置”和“收入由谁驱动”。
- 能判断标的所处环节的分工生态和周期强弱属性。

### 模块 7：Memory / Storage 深水区

核心问题：为什么不能听到“存储缺货”就一锅端？

必须懂的概念：
- DRAM
- HBM
- NAND
- eSSD
- HDD
- data center SSD
- checkpoint
- KV cache
- 带宽 vs 容量
- commodity vs 结构性短缺

案例：
- MU 是 DRAM + HBM 叙事。
- SNDK 是 NAND + eSSD 叙事。
- WDC / STX 是 HDD 和冷存储逻辑。

练习题：
- 听到“memory shortage”时，如何判断它指的是 HBM、DRAM、NAND、eSSD 还是 HDD？
- 为什么 eSSD 的利润弹性可能来自 mix shift，而 HBM 更偏技术壁垒溢价？

Memo 输出能力：
- 能写清楚存储标的到底卖的是带宽、容量、企业盘，还是冷归档。

### 模块 10：周期、库存与供需拐点

核心问题：如何判断现在是周期上行、顶部、下行，还是结构性重估早期？

必须懂的概念：
- Secular

- 库存
- 去库存
- 补库存
- 牛鞭效应
- lead time 缩短
- 设备订单领先
- 供给释放

案例：

- 存储 2023 减产到 2026 供给紧张。
- 设备订单早于下游财报改善。
- 云厂商从现金流内投资转向发债融资，是资本开支可能减弱的前置信号。
- 高性价比模型打价格战，会加大头部模型公司的亏损和融资压力，动摇整个算力需求的地基。

练习题：
- 为什么“需求强劲”可能同时是利好和风险？
- 一个行业供给恢复需要 24-36 个月，对持有期有什么影响？
- 云巨头资本开支增速从 60% 回落到 30%，绝对值仍在增长，这对产业链不同层分别意味着什么？

Memo 输出能力：
- 能写出“周期位置”和“买点风险”。
- 能列出证伪整个 AI 需求地基的前置信号并持续跟踪。

### 模块 11：财务语言与估值

核心问题：如何把产业故事翻译成收入、利润、现金流和估值？

必须懂的概念：
- 收入
- 毛利率
- 营业利润率
- 折旧
- CapEx
- FCF
- 库存
- 应收账款
- 经营杠杆
- PB
- PE
- DCF
- 周期股重估

案例：
- NAND ASP 上涨如何放大毛利率和营业利润率。
- 存储从 PB 定价向 PE 重估的争议。
- Intel 代工良率承诺如何影响利润。

练习题：
- 为什么固定成本高的行业涨价后利润弹性可能远大于收入弹性？
- 什么情况下一个公司应该用 DCF，什么情况下只是交易筹码？

Memo 输出能力：
- 能写出估值逻辑、利润弹性和估值风险。

### 模块 12：美股 / A 股 / 港股 / 台韩日市场差异

核心问题：同一个 AI 产业逻辑，为什么在不同市场要用不同验证和折扣？

必须懂的概念：
- 美股 earnings call
- 10-Q / 10-K
- 回购
- A 股映射
- 国产替代
- 集采压价
- 港股估值折价
- 台股供应链
- 韩股存储
- 日股材料设备
- 流动性折扣
- A 股资金结构
- 回购文化差异

案例：
- 美股更重财报、现金流、管理层指引。
- A 股会把海外热点映射到供应商、供应商的供应商。
- 台股、韩股、日股常是供应链真实源头。
- A 股缺少成熟配置型资金，只剩短期博弈资金和长期锁定资金，AI 与非 AI 估值分化比成熟市场更极端。
- 美股、韩股龙头按资本效率持续回购；A 股公司凭管理层对股价的主观判断决定是否回购。

练习题：
- 同一个 HBM 逻辑，在美股、韩股、A 股分别怎么找标的和风险？
- A 股“国产替代”为什么方向确定不等于个股能赚钱？

Memo 输出能力：
- 能写出跨市场可比公司和不同市场的风险折扣。
- 能解释标的所在市场的资金结构对估值波动的放大或抑制作用。

### 第四层：研究输出层

目标：从看懂别人观点，转向自己研究、自己判断、自己复盘。

### 模块 13：证据链与研究流程

核心问题：如何从“听别人说”变成“我验证过”？

必须懂的概念：
- Earnings call
- 10-Q
- 10-K
- 管理层语气
- fireside chat
- 设备订单
- LTA
- 预付
- 客户集中度
- 存货
- 同行交叉验证
- AI 对比多年发言变化
- 买方视角 vs 卖方视角
- 内部人动向

案例：
- ER 和电话会讲故事，10-Q 把数字落到正式报表。
- 管理层从 aggressive capex 变成 disciplined capex 的含义。
- 用 CEO 讲话变化寻找投资假设。
- 研报多是卖方立场：要主动找负面证据、看内部人增减持，警惕联合做局和信息污染。

练习题：
- 给一段 earnings call，标出哪些是叙事，哪些是可验证证据。
- 如何用 10-Q 检查管理层说的需求强劲是否可信？

工具清单：
- 法定披露文件：SEC 官方检索（10-K、10-Q、13F、内部人交易）。
- 财报电话会转录与分析师问答：Seeking Alpha 等转录站。
- 公司基本面与行情数据：Alpha Vantage、Polygon、Alpaca。
- 图表工具：TradingView。
- 管理层访谈聚合与公司投资者关系页面。

Memo 输出能力：
- 能列出证据链、未验证假设和下一步研究清单。

### 模块 14：投资 Memo 与个人系统

核心问题：如何把研究变成一篇可复盘、可执行、能承认错误的投资 memo？

必须懂的概念：
- 投资 thesis
- 反方观点
- catalyst
- kill criteria
- core / satellite
- 仓位上限
- holding period
- 风险承受力
- 风险偏好
- 复盘标准
- 仓位×策略匹配
- 资金效率
- FOMO

超出产业研究范围的个人资金建议：
- 仓位指占家庭可投资资产的比例：低仓位可配激进策略，高仓位必须配保守策略，双激进是赌博。
- 个人投资者没有排名和赎回压力，应抓赔率与胜率、主动放弃资金效率；不借钱投资、不倒金字塔加仓。
- 三个不需要深度研究能力的办法：可转债、约 20 只分散高股息被动高抛低吸、确认不死的行业从前高至少腰斩后分批买。
- 不要 FOMO：每隔一两年都有新主线，等板块企稳、看得到空间再参与。

最终 memo 模板：
1. 标的是什么。
2. 处在产业链哪一层。
3. AI 需求如何传导到它。
4. 四铁律打分。
5. 关键证据。
6. 美股 / A 股 / 港股 / 台韩日可比公司。
7. 周期位置。
8. 财务与估值判断。
9. 反方观点。
10. 什么情况证明我错了。
11. 仓位和持有周期。
12. 最终结论。

练习题：
- 选一个陌生 AI 半导体公司，用模板写一页 memo。
- 写出你最可能错的三个地方，以及对应的验证信号。
- 用仓位×策略矩阵给自己当前的组合定位，判断是否匹配。

Memo 输出能力：
- 能独立完成一篇可复盘的投资 memo。
