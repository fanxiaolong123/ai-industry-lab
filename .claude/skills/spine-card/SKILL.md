# 骨架卡生产标准

## 定位

骨架卡是四层学习体系的第一接触面：零基础学习者最先读到的东西，为整个领域搭架子。全部 15 张卡的标题和一句话概括在 `Learn Plan - AI半导体投资.md` 的「骨架卡清单」；定稿在各 `output/spineNN.html`。

金标准：`output/spine01.html`（**供给慢，需求快**）。动手前先完整读一遍它。

## 三条最高原则

1. **一张卡只承载一个模型。**上限是概念数，不是字数——篇幅以把这一个模型讲透为准（读完约 10-15 分钟）。写作中若发现需要展开第二个模型，停下来：要么引用其他卡（「卡 X 展开」），要么把它交给手册页。这条原则的来历：第一版卡片因为「就这几句话」被否（模型没讲透），而手册页曾因几十个概念一次咬合被否（云里雾里）——骨架卡的存在意义就是站在两者之间：**单模型，但五脏俱全**。
2. **查证纪律与全项目一致。**锚点数字写之前用 WebSearch 核验一手来源（年报、电话会转录、官方统计、官方公告），或沿用项目里已核验的来源（见 output/m02.html 页脚清单）；查不到就换锚点或明确标注不确定。凭记忆的数字不上卡。
3. **核心结论必须挂高质量讨论文章。**每张骨架卡的核心结论（那张卡承载的模型/断言）**必须挂至少 2 篇高质量讨论文章**——不是硬事实来源（年报/10-K/8-K/电话会转录/官方公告/官方统计），是**观点讨论来源**：分析师深度报告、学术论文、权威媒体深度调查、圈内知名投资人的公开表述/股东信/演讲/访谈，具有独立论证过程（不是转述摘要）。作用是把结论与观点市场连起来——读者点开一篇文章就能看到别的深度声音，判断这个结论在观点市场上有没有被系统讨论过。找不到高质量讨论文章的结论，是没进入观点市场的孤立断言，通常意味着结论表述过窄或过俗——回头看结论本身的定位。

## 六段固定结构

依次写，段段不缺：

1. **现象钩子**——一个真实的、带日期的、有反差的现象（例：同一批公司一年内从抢货到砍单）。让读者先看见「怪事」，末尾一句话点出本卡模型。不从定义开头。
2. **机制推导**——回答「为什么会这样」，推到读者能自己复现为止。每一步要么可查证，要么标注口径（如「行业通行口径」「专家判断」）。分点写，每点一个原因。**段末必须有一小节「解释范围」**：本卡结论解释的现象，现实中常是几条机制叠加造成的——明说本卡只讲其中一条、另外几条在哪张卡（数量按需，不定死）、几条怎么一起用（给一个具体场景做示例）。不写这节，读者会拿一条机制去解释整个现象。
3. **真实锚点**——机制落在已核验的真实数字和事件上（带来源与日期）。锚点的作用是让模型「有重量」，至少两个。
4. **投资含义**——输出「第一反应问句」形式的条件反射：看到某类新闻，第一反应问什么、去哪查。这是卡片的可用性所在——读者以后靠这几个问句使用这张卡。
5. **边界**——这张卡什么时候不适用；用「卡 X 展开」预告相关卡，形成卡间连接。没有边界的模型会被滥用。
6. **练习两道**——①复述题：合上卡片讲给完全不懂的人（注明时长量级）；②推演题：用**已核验的真实事实**做前提（前提本身要先查证），让读者用本卡机制推一个卡里没有直接给答案的结论，附提示、不附标准答案。

结尾加一行「卡 N 来源」：列出本卡全部锚点的来源名与日期。

## 徽章可点击（强纪律）

每一个徽章都必须是可点击的 `<a>`，点击直接跳向该条断言的核验源头。**语法与 manual-page skill 一致**：

- 优先直链一手 URL：`<a class="badge badge-verified" href="https://…" target="_blank" rel="noopener">已核验 · 来源名+日期</a>`
- 无直链时锚点到页脚 `.src` 的 `<li id="src-N">`：`<a class="badge badge-expert" href="#src-3">专家判断 · 通行口径</a>`
- 页脚 `.src`（骨架卡页也放来源清单，命名"卡 N 来源"）每个 `<li>` 必须有 `id`；外链一律 `target="_blank" rel="noopener"`。

`<span class="badge …">` 一律返工。

## 语言纪律

零基础白话：假设读者不认识任何行业词。新术语当场用一句人话解释（「交期（下单到提货的等待时间）」）。中文为主，专有名词保留原文。禁止翻译腔死译、名词化比喻、抽象拔高、AI 套话。语气是讲给聪明朋友听，不是播新闻。

## 产出与登记

- **HTML 页 `output/spineNN.html` 是唯一定稿真源**（2026-07-08 用户判定）。骨架卡页面按六段结构写，卡页可为核心机制配直观化组件（标准见 manual-page skill 第三步 A 类）。
- Learn Plan 里**不再放**骨架卡教学文本节——HTML 页是定稿真源，Markdown 里再抄一份是重复维护负担。
