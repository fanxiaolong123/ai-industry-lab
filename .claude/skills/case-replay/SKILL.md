# 案例页生产标准

## 定位

案例页是四层学习体系的第三层（骨架 → **案例** → 手册 → 实盘）：读者过完 15 张骨架卡之后进入，用真实历史决策把模型练成体感。**案例训练推理链，不训练股价**。首批 12 个的案例池、六要素、生产流程与验收清单在 `Learn Plan - AI半导体投资.md` 的「案例主线：质量标准与生产流程」节；定稿在各 `output/caseNN.html`。

金标准：`output/case01.html`（待建 · 台积电 N3 稀释 2024-01-18 电话会）。case01 完成 QA 后回填此段（含实际路径与"先完整读一遍它"的硬要求）。

## 三条最高原则

1. **信息集严格切在判断日。** 判断点之前只放当时可得的信息，严防事后信息泄漏。每一条数字、事件、原话引文、市场情绪都必须带明确时间戳，出现晚于判断日的任何信息就把这条移到"事后对照"节。这条纪律与骨架卡"数字查证"平级、但更严——骨架卡查证的是数字对不对，案例查证的是**时序**对不对。用了事后数据，读者本可以事先算出的判断变成"上帝视角"，整个案例的教学价值归零。发布前必须 grep 判断日之后的所有日期，逐条确认不出现在信息集节里。

2. **推理链五要素齐全：定位 → 机制 → 数量化 → 反方 → 可证伪判断。** 这是模块 14 memo 模板的缩微版。任一要素缺失，把案例降级到「新闻回顾」而不是「决策复盘」。反方必须是**真正对抗**——列 2-3 条最有力的反方论证，每条能独立推翻本判断，不是「当然也可能失败」式套话；可证伪判断必须具体到**数字与时点**，「毛利率 Q2 2024 未回到 53% 以上就承认错」这种形态，不是「情况恶化就承认错」这种空话。

3. **事后对照必须区分推理质量与结果运气。** 案例的核心训练点不是「股价对不对」，是**推理链本身立不立得住**：推理对且结果对（能力）／推理对但结果错（运气不佳）／推理错但结果对（运气偶合）／推理错且结果错（判断失败）。案例只训练前者。混淆这两者是学习者最常见的失败模式——「专家说涨结果真涨了 = 推理对」是错的，要看**推理链本身**是否建立在扎实的机制与数字上。教训段必须写清本案例落在哪一格、为什么。

## 七段固定结构

依次写，段段不缺：

1. **决策现场**——一句 thesis + 判断日 + 决策方（真实人物、类别或虚构分析师，如「2024-01-18 台积电 Q4 2023 电话会散会当晚，一名跟踪台积电的美股买方分析师面对的问题」）。三行说清读者要在什么时点、以什么身份、面对什么问题下决策。不引言、不铺垫，直入现场。

2. **判断点前的信息集**——严格切在判断日之前。四类信息必须齐全，每一条带日期与徽章：
   - **时间线**：一张时序 SVG 或结构化时间线，红竖线切割判断日；之前是「已知」、之后节隐藏（不在本节出现）
   - **已知数字与事件**：财报数字、指引、订单信号、宏观数据；每条注明来源日期
   - **已有原话引文**：管理层公开表述、分析师报告观点、卖方一致预期——**逐字原话**，不是编者转述
   - **市场情绪与共识**：当时的股价位置、卖方一致预期、散户话题、看多看空的主要论点
   
   信息集只放读者判断前需要知道的、不放事后才知道的。宁可信息集显得"少"，也不能塞进任何未来信息。

3. **学习者动作 · 先下判断**（第一个动作点）——3-5 个具体问题，读者本地写下答案再展开专家答案。问题形态："如果你是当天做决策的分析师，你会：① 判断产业链哪一层最先受影响 ② 给一个可证伪信号 ③ 给自己一个持有周期 ④ 列 2 条反方论证"。用 `<details>` 隐藏区包住专家答案，`<summary>` 明确写「先合上笔记本自己写完再展开」。

4. **专家推理链**——五要素依次展开，每部分标题清晰：
   - **① 定位**：本决策涉及的公司在产业链哪一层、由谁的资本开支驱动、上下游是谁；引用相关骨架卡编号（如「见卡 4：出货 = 最短板」）
   - **② 机制拆解**：当时的核心矛盾是什么、几条机制怎么互动；每条机制引用骨架卡编号，避免重复讲卡里已讲的内容
   - **③ 数量化算账**：从当时可得数据推出关键数字（毛利率、稀释幅度、回正时长、订单弹性等）；**此处配"先自己算再看"的隐藏组件**（第二个动作点）——给读者具体输入数据与计算问题，专家算法用 `<details>` 隐藏
   - **④ 反方观点**：2-3 条最有力反方，每条独立成段，写清"如果这条成立，本判断怎么翻"；反方必须真对抗，形式主义反方一律不合格
   - **⑤ 可证伪判断**：明确到"哪个数字变到哪个值、在什么时点，我承认判断错"；至少 2 条可证伪信号，每条配跟踪频次（"每季度电话会""每月出货数据"等）

5. **反方与失效分析**——补第 4 节反方观点之外的失效边界：外部冲击（政策、地缘、疫情级黑天鹅）能让整个推理归零吗？除了反方直接反对，还有哪些"结构性外部条件变化"能推翻本判断？这一节独立于反方观点，看的是"不可控外因"。

6. **事后对照与归因**——**必须与前面严格切开**（视觉、位置、时间戳）：
   - **实际发生了什么**：判断日之后的事实序列，每条带日期与徽章
   - **数字兑现对照**：推理链的每条数量化算账，实际数字与判断日估算的对比（表格形态最佳）
   - **四象限归因**：2×2 矩阵（推理对错 × 结果对错），明确落在哪一格并解释为什么；矩阵可用 SVG 或纯 CSS 呈现，配一句"看什么"引导
   - **教训**：从本案例能带走的**可复用规则或警示**——不能只是"事后看很清楚"式空话；教训要能挂回骨架卡（"这次强化了卡 X 的边界条件"）或指出骨架卡漏了的场景

7. **来源清单**——全部一手来源带日期，页脚 `.src` 一览可审计，每个 `<li>` 带 `id`。

结尾加一行"案例 N 来源"标题。

## 徽章可点击（强纪律）

完全继承 spine-card 和 manual-page 的徽章规则，不重述。要点回忆：

- 三类：`badge-verified`（已核验 · 一手 URL）/ `badge-expert`（专家判断 · 附出处）/ `badge-assume`（估算 · 估算方+年份）
- 每一个都必须是可点击的 `<a>`；`<span class="badge …">` 一律返工
- 有直链的 `href` 一手 URL + `target="_blank" rel="noopener"`；无直链的锚点到页脚 `.src` 的 `<li id="src-N">`
- 案例页对**时间戳的可核验性**要求更高：每个信息集中的事实必须能追回到"判断日之前"的具体来源与日期，读者需能验证"是当时可得的"

## 交互组件

案例特有的四个组件，都是必配（非可选）：

1. **时间线切割组件**（A 类 · 直观化）——SVG 时序图，红竖线（`--danger`）明标判断日；之前的事件用墨色实心点、之后的用灰色空心点（如果本节不显示只做视觉引导）。每个点悬浮出徽章可跳转。配「看什么」引导语：「判断日左侧是当时可用信息，右侧留到事后对照节展开」。

2. **判断点隐藏与揭晓**（B 类 · 推理组件）——用 `<details>` 隐藏专家推理链。`<summary>` 严格写「合上笔记本先写完 3-5 个问题的答案，再展开专家推理」。展开后不给单一"标准答案"、给专家的完整推理路径与几个可能的答案光谱（"这条问题合理的答案在 X-Y 之间；如果你写的是 Z，去反方观点节看这条为什么被否"）。

3. **中途算账组件**（B 类 · 推理组件）——在推理链的"数量化算账"环节。给读者一个具体输入数据 + 计算问题（"N3 稀释 3-4pt 毛利率、N3 占公司总产能 4%，请算全公司毛利率被拉低多少"）；先隐藏专家算法用 `<details>`。展开后必须给公式 + 数字来源 + 结果 + 一句「看什么」（"注意分母是总产能不是先进制程产能"）。

4. **四象限归因矩阵**（A 类 · 直观化）——2×2 矩阵（横：推理对/错；竖：结果对/错），本案例的判断落一个高亮点，配一句解释为什么落在这一格。矩阵不必华丽——一个 CSS grid 4 格 + 一个绝对定位点就够。配「看什么」引导语：「这个位置说明推理质量与结果之间的关系，训练案例只关心横轴」。

组件通用纪律（继承 manual-page）：
- 每个组件配一句「看什么」引导语
- 教学性简化必须标注（"四象限归因是简化框架，真实决策往往落在灰色地带"）
- 纯原生 SVG/CSS/JS；动画尊重 `prefers-reduced-motion`；键盘可达

## 信息集切割的实操细则

案例区别于骨架卡/手册页的核心难点是**时序纪律**。以下是常见泄漏形态与查法：

**泄漏形态**：
- **数字泄漏**：引用了判断日之后才披露的数据（"当时可以推出 FY24 毛利 56.1%"——56.1% 是 2025-01 才有的年度数字，判断日不可知）
- **叙事泄漏**：用了后来的语气描述当时的情绪（"当时市场没意识到 N3 稀释"——这个"没意识到"是事后归纳）
- **归因泄漏**：把事后归因塞进"当时的机制拆解"（"当时大家没看到 AI 需求即将爆发"——判断日 2024-01 时 AI 需求已经在发生了，不能事后倒放）
- **对比锚泄漏**：用了"后来我们知道的"参照点（"当时 N5 毛利已经是 45%"如果 45% 是后来某季数据，就是泄漏）

**查法**：
- 发布前 grep 判断日之后的所有月份/季度/年份（如判断日 2024-01-18，grep "2024-Q2""2024-03""2024-04"..."2025""2026" 等），逐条确认不出现在信息集与推理链节里
- 每条数字问自己："这个数字在判断日之前**公开可得**吗？"能可得则留、不能则移到事后对照
- 每条引文问自己："这句话在判断日之前**已经说过**吗？"未说过则移

## 语言纪律

继承 spine-card 和 manual-page：零基础白话，专有名词首次出现配一句解释；不用翻译腔、名词化比喻、抽象拔高、AI 套话、营销话术。案例特有的语气纪律：**不能是"讲故事"的语气**——不用"当时市场情绪高涨"这类小说化叙述，用"XX 日 SOX 指数收于 X 点，卖方一致预期 Y"这类可查证的具体表述。案例读起来应像**一份复盘 memo**，不是财经报道。

## 修订与二次审阅

案例页评审沿用 `.claude/skills/page-review/SKILL.md` 的模式。案例特有的评审角度：

- **时序纪律**：信息集节是否严格切在判断日之前（这是骨架卡/手册页评审不涉及的独立维度）
- **反方真对抗**：反方观点是形式主义还是真能推翻本判断
- **推理与结果分离**：事后对照节是否混淆了推理质量与股价结果
- **教训可复用性**：教训是"事后看很清楚"空话，还是能挂回骨架卡或指出骨架卡漏洞的可复用规则
